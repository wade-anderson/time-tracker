# TimeTracker App - User Requirements

1. **Core Application**: Build a simple, clean, minimal, and mobile-friendly time tracking web application.
2. **Data Entities**: Allow users to manage and add Customers, Projects, and Invoices.
3. **Basic Task Logging**: Allow users to log tasks including a description, selected project, selected invoice, start time, and end time.
4. **Time Summaries**: Provide summaries of total time spent per project, per day, and per week.
5. **Report Generation**: Ensure all printed reports are accurate, professional, and legible. Printed reports must explicitly feature page numbering (e.g., "Page 1") at the top right, but these numbers must remain completely invisible on the screen versions of the reports. The system must also prevent content truncation across page bounds, avoid breaking project data across pages, and hide UI buttons (like the print button) during printing.
6. **Invoice Details**: Display total monetary amounts directly on invoice cards.
7. **Project Statuses**: Projects are automatically marked as Active when created. Users can edit a project to mark it as Completed. When logging a new task, only Active projects can be selected.
8. **Task Editing Improvements**: When editing an existing task from the Manage Tasks screen, allow the user to change both the Project and the Invoice associated with the task.
9. **Customer Requestors**: Customers must have a list of "Requestors." These requestors appear as a selectable dropdown list when logging a task for that customer's projects.
10. **Requestor Visibility**: Display the selected Requestor for each task on both the Manage Tasks view and the Recent Tasks list.
11. **Dashboard Simplification**: The main dashboard should be clean; the "Project Totals" card should be removed, and the "View Report" button should be placed conveniently within the main sidebar.
12. **In Progress Tasks**:
    12.1. **Initiation**: An "in-progress" task is initiated if a task is logged where the start time and end time are identical. If a task spans any duration (end time != start time), it is immediately treated as a completed task.
    12.2. **Quick Start Button**: Provide a "Create In Progress Task" button on the Log Task card that instantly creates an in-progress task using the current time for both start and end.
    12.3. **Dashboard Card**: Display an "In Progress Task" card (above the Active Invoices card) that surfaces the active task alongside a live elapsed-time timer based on the start time and current time.
    12.4. **Completion**: Provide a "Complete Task" button on the In Progress card that sets the end time to the current moment, calculates the final duration, and marks it as complete.
    12.5. **Deletion**: Provide a Delete capability (an '×' button) on the In Progress card to instantly discard and delete the in-progress task.
13. **User Interface Accessibility**: When a popup/modal window is active, pressing the ESC key must cancel and close it.
14. **Data Storage Mechanism**: Application state must be persisted asynchronously using IndexedDB to support larger datasets, ensuring automatic migration for any legacy data residing in localStorage.
15. **Data Deletion Safeguards**: The system must explicitly ask for user confirmation before executing the deletion of any task, invoice, customer, or project to prevent accidental data loss.
16. **Automatic Backups**: The system must automatically generate and download a comprehensive JSON backup of all application data immediately after a user successfully logs a new task or initiates an in-progress task.
17. **Invoice Sorting**: The list of invoices must be sorted so that active invoices appear at the top, followed by the remaining invoices sorted by their submission date (most recent first).
18. **Version Display**: Display the application version number (e.g., "v1.0.0") on the top right of the main page header.
19. **XSS Protection**: Ensure DOM text is not reinterpreted as HTML by escaping meta-characters in all user inputs before rendering them onto the page.
20. **In Progress Task Editing**: Allow editing of in-progress tasks directly from the dashboard. When editing an in-progress task, the start and end time inputs must be disabled to prevent accidental timer modifications, and the "Task updated!" popup should be suppressed upon saving.
21. Projects
    21.1. **Project Statuses**: Projects are automatically marked as Active when created. Users can edit a project to mark it as Completed. When logging a new task, only Active projects can be selected.  Active Projects appear on the top of the list.  Completed Projects appear at the bottom of the list.  Completed Projects can be edited to change the status to Active.  Completed Projects can be deleted.  Active Projects can be deleted.
    21.2. Put a button on the each project in the project list to toggle between Active and Completed status.
